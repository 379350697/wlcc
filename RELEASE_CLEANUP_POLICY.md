# RELEASE_CLEANUP_POLICY

## 目标
在不影响正式版代码与主链行为的前提下，清理 release 仓中的路径 hardcode、样本任务名硬编码与旧结果残留。

## 原则
1. 不改坏正式运行脚本主逻辑。
2. 优先参数化，避免继续写死样本名。
3. 对历史结果文件，允许重写为 release 路径或明确标注为样本结果。
4. 对同步辅助脚本，允许保留 research 来源说明，但不让其误导为运行时依赖。
5. 对 demo / sample / fixture，尽量集中为显式样本用途，而不是伪装成正式主链默认输入。

## 清理优先级
### P1 必清
- 正式运行脚本里的固定 task id
- 正式运行脚本里的 research/local 绝对路径
- 结果文件中的 research 路径误导

### P2 建议清
- 测试脚本里的样本 task 名写死
- 旧 phase2 运行态结果直接暴露在正式说明里

### P3 可保留但需说明
- 同步脚本中的 research 来源路径
- phase2 样本 fixture 本身
