# MEMORY_LAYERING_PLAN

## 目标
按执行文件 Task 5，明确把以下信息分层：
- 长期规则
- 项目事实
- 任务状态
- 会话摘要

## 分层结构

### 1. 长期规则层
来源：
- `SOUL.md`
- `AGENTS.md`
- `USER.md`
- `MEMORY.md`

特点：
- 变化慢
- 不应被项目临时状态污染

### 2. 项目事实层
来源：
- `README.md`
- `STATUS.md`
- `DECISIONS.md`
- `TASKS.md`
- `INCIDENTS.md`

特点：
- 项目事实优先于聊天摘要
- 是项目判断的一等依据

### 3. 任务状态层
来源：
- `.agent/tasks/*.md`
- `.agent/resume/*.md`

特点：
- 生命周期与任务绑定
- 表示当前执行态
- 可恢复

### 4. 会话摘要层
来源：
- `tests/*.md`
- `SKILL_VALIDATION_ROUND1.md`
- 后续 compact / handoff 产物

特点：
- 可压缩
- 可替换
- 不覆盖项目事实

## 读取优先级
1. 长期规则（约束）
2. 项目事实
3. 任务状态
4. 会话摘要
5. 聊天上下文

## 兼容策略
- 不删除旧结构
- 新结构优先读
- 缺失时降级到旧文件或已有摘要
- 摘要不得覆盖事实源
